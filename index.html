Molecularity

This game is a group created game called Molecularity.The game is about changing a cubes properties using a multi-tool to solve levels, a play through can be found here on vimeo(video below). It was created for a university assignment. It is done using DirectX and C++. Some of the features of the game are:

  • A main menu
  • 3 levels
  • Credits
  • A working settings menu
  • A hud for the tool information
  • Built with design patterns

The multi-tool:

  • Convert-changes the material which affects mass/density​
  • Resize- enlarges or shrinks a cube which changes its hitbox and mass​
  • Bounce- whether the cube bounces​​
  • Magnetism- either brings a single or all the cubes to the player​
  • Conductive- changes a cube’s conductivity​

I created the UI used in the game except for the DIMGUI used for the debug mode. I also created a json loader file with rapid json to load our text and level data. The base of our tool class in the code was created by me.

Other team mebers:

  • Kyle Robinson- lead engine programmer and level programming
  • Ben Shaw-engine programmer and sound
  • Daniel Wilson- engine programmer and 3D model creation
  • Jacob Dexter- physics programmming

You can find it at:Molecularity

FGGC- sem1 and sem2

This is a engine created for a my university assignment. It has been created with C++ and DirectX 11.

Sem1 key features:

  • Shaders in HLSL
  • Phong and Gouraud shading
  • OBJ file loader
  • DDS file loader
  • Picking
  • simple movement
  • mutiple cameras:
    • Static cameras
    • First-person camera
    • Third-person camera
  • Json file loader useing nlohmann json parser
  • A skybox using cube mapping

video

Sem2 key features:

  • Physics- including: particle physics for object movement and rigid body physics for rotations
  • particle physics uses:
    • Velocity
    • Acceleration
    • Drag- Laminar and Turbulent
    • Friction
    • net force accumulator
    • other forces such a gravity and thrust
  • Simple collision detection
  • Collision response
  • Mass particle system with rain and smoke
  • Terrain generation useing a hight map
  • Input class using win32 messaging system
  • Built with design patterns
