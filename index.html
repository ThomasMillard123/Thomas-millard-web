Firelight

This game is a group created game called Firelight. It is a survival game. The engine that was created was called Firelight and a audio engine was created with FMOD called Snowfall. An editor was created to help to develop on the engine but for the most part the engine is designed to work for developing in code. It was created for a university assignment in a group, and it is coded using DirectX 11 and C++. Some of the features of the game and engine are:

  • Entity component system
  • Input with key bindings
  • Event system
  • AI
  • Resource system
  • Crafting
  • Items
  • An inventory system
  • Animation with an editor
  • Audio- engine and sound in the game
  • Combat

I created and developed:

  • The UI development system in the engine
    • Use the ECS to create widgets out of components
    • Has the ability to create widgets that are pressable, to be hovered, draggable and to have on leave functions
    • Widgets can also have a component that gives them to have the ability to contain children widgets
    • The widgets can scaled by their parent or the canvas
    • The widgets are also scaled to the window size
  • The method to get input to the game and engine
    • Uses window messages to get access to the keyboard and mouse input
    • Uses XInput to get input from a controller
    • Using events this data is communicated to other parts of the engine, like the key binder
  • The inventory system
    • Uses groups to allows of related inventories to be held together
    • Holds the entityIDs of items
    • The items can be held in a stack according to the item data
    • The slots and the background of the inventories can be customised
    • Data can be added to the slots to limit what items can be inserted into it

Molecularity

This game is a group created game called Molecularity.The game is about changing a cubes properties using a multi-tool to solve levels, a play through can be found here on vimeo(video below). It was created for a university assignment. It is done using DirectX and C++. Some of the features of the game are:

  • A main menu
  • 3 levels
  • Credits
  • A working settings menu
  • A hud for the tool information
  • Built with design patterns

The multi-tool:

  • Convert-changes the material which affects mass/density​
  • Resize- enlarges or shrinks a cube which changes its hitbox and mass​
  • Bounce- whether the cube bounces​​
  • Magnetism- either brings a single or all the cubes to the player​
  • Conductive- changes a cube’s conductivity​

I created the UI used in the game except for the DIMGUI used for the debug mode. I also created a json loader file with rapid json to load our text and level data. The base of our tool class in the code was created by me.

Critical evaluation of current weather systems FYP

This is my final year project; it is looking at the how weather is created in games and how it is used. During the exploration of other games, I saw there was lots of methods to create weather systems. These different methods depended on what the game wanted to achieve or the hardware limitations of then games chosen platform. For example, a simulator game would want the most complex and realistic weather possible, Microsoft flight simulator does this by using real world weather.

In the project, I looked at how real weather works and how this could be used in a games weather system. I also looked at how the more realist the weather system wants to be the more resources it will use. This also looked at how a weather system needs to be more immersive than realistic, so the player feels like they are in the world. The artifact created was to use real world concepts to create a weather system.

The artefact created used airmass and a grid to hold data, like temperature at different hights and dew point, to create a weather. This system can currently create fog, precipitation (snow, hail, and rain) and clouds. The air masses work well in creating weather fronts for where the weather effects appeared. However, the gid limited how the weather effects were represent as it held a lot of data needed. If the project was going to be improved the removal of the gid would be done in favour of the air mass holding more data or use of another method. The artefact has been created with DirectX 12 with the use of miniEngine.

In the end other method currently in use could be combined with airmass and weather fronts to create better flowing weather.
